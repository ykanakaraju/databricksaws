1. Create an S3 bucket

	Name: ykr-databricks-metastore
	
2. Create an IAM role (with federated access)

	** Databricks will have a role. That role will assume a role in our account (that we are creating now)
	   We can now control what databricks can do with our account using this role.
	   
	- Trusted entity type: Custom trust policy
	- Role Name: YKRDatabricksMetastoreRole
	- Add the following JSON snippet in the 'Trust relationship' tab and replace the place holders
	- Finish creating the role (no policies are attached yet).

{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "Statement1",
            "Effect": "Allow",
            "Principal": {
                "AWS": [
                    "arn:aws:iam::<YOUR_AWS_ACCOUNT_ID>:role/<THIS_ROLE_NAME>",
                    "arn:aws:iam::414351767826:role/unity-catalog-prod-UCMasterRole-14S5ZJVKOTYTL"
                ]
            },
            "Action": "sts:AssumeRole",
            "Condition": {
                "StringEquals": {
                    "sts:ExternalId": "<DATABRICKS_ACCOUNT_ID>"
                }
            }
        }
    ]
}


3. Create a Policy and attache it to the above role.

	Policy Name: YKRDatabricksMetastoreS3Policy
	Add the following JSON snippet and replace the place holders
	Attach this policy to the role created earlier	
	
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Action": [
                "s3:GetObject",
                "s3:PutObject",
                "s3:DeleteObject",
                "s3:ListBucket",
                "s3:GetBucketLocation",
                "s3:GetLifecycleConfiguration",
                "s3:PutLifecycleConfiguration"
            ],
            "Resource": [
                "arn:aws:s3:::<BUCKET_NAME>/*",
                "arn:aws:s3:::<BUCKET_NAME>"
            ],
            "Effect": "Allow"
        },
        {
            "Action": [
                "sts:AssumeRole"
            ],
            "Resource": [
                "arn:aws:iam::<YOUR_AWS_ACCOUNT_ID>:role/<AWS_IAM_ROLE_NAME>"
            ],
            "Effect": "Allow"
        }
    ]
}


 4. Add a bucket policy to declare which roles have access to the bucket.
 
	Open the bucket created in Step 1 and add the following bucket policy.
	Replace the place holders.
	
{
	"Version": "2012-10-17",
	"Statement": [
		{
			"Sid": "Statement1",
			"Principal": {
				"AWS": "arn:aws:iam::<YOUR-AWS-ACCOUNT-ID>:role/<ROLE-NAME>"
			},
			"Effect": "Allow",
			"Action": [ "s3:*" ],
			"Resource": [
				"arn:aws:s3:::<BUCKET>/*",
             	"arn:aws:s3:::<BUCKET>"
			]
		}
	]
}

 5. Login your your AWS account console and create a workspace
 
 6. Create a metstore and attach the workspace to this metastore
 
	- Click on the 'Catalog' and 'Create metastore' button
	
	- Name: YKRDatabricksMetastore
	  S3 bucket path: Bucket created in step 1
	  IAM role ARN: Role created in step 2
	  
	- Create the metastore and attach the workspace.
